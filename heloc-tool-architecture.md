# Above All CRM — Carbon HELOC Tool Architecture (v12)

## File Overview

Single HTML file (~2,180 lines) with embedded CSS and JavaScript. No dependencies except html2canvas CDN. Deployable on any static host (Cloudflare Pages, GHL funnels, direct file).

**WARNING**: When deploying on Cloudflare Pages, the platform may inject email obfuscation scripts (`/cdn-cgi/scripts/*/email-decode.min.js`) that break JavaScript execution. Always strip Cloudflare artifacts before editing.

## Security: Password System

Two-tier password protection using base64-encoded passwords:

```javascript
const SETTINGS_PASSWORD = atob('V0NMMjAyNg==');   // WCL2026
const ADMIN_PASSWORD = atob('R29vZGxpZmUyMDI2'); // Goodlife2026
```

| Password | Access Level | What It Unlocks |
|----------|-------------|----------------|
| Settings (WCL2026) | Basic admin | Rate editing, integration config, email templates |
| Admin (Goodlife2026) | Full admin | All Settings + lead parser, advanced tools |

## Known Issues & Fixes

### Cloudflare Email Obfuscation
When hosted on Cloudflare Pages, emails get replaced with obfuscated `data-cfemail` attributes and a decode script is injected. This breaks the HTML structure.

**Fix**: Strip all `/cdn-cgi/` references and restore plain-text emails. Use Python regex for reliable replacement:
```python
import re
content = re.sub(r'<script[^>]*data-cfasync[^>]*>.*?</script>', '', content, flags=re.DOTALL)
content = re.sub(r'<a[^>]*class="__cf_email__"[^>]*>[^<]*</a>', 'Eddie@WestCapitalLending.com', content)
```

### File Truncation
Large HTML files can get truncated during editing. Always verify:
```bash
# Check file ends properly
tail -5 file.html  # Should end with </script></body></html>

# Verify brace matching
grep -o '{' file.html | wc -l  # Must equal:
grep -o '}' file.html | wc -l

# Verify parenthesis matching
grep -o '(' file.html | wc -l  # Must equal:
grep -o ')' file.html | wc -l
```

### str_replace Failures
Complex JavaScript with unicode characters and backtick template literals often fails with str_replace. Use Python file operations instead:
```python
with open('file.html', 'r') as f:
    content = f.read()
start = content.find('target_string')
# ... modify content ...
with open('file.html', 'w') as f:
    f.write(content)
```
